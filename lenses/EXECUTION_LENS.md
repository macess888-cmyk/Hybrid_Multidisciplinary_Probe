# Execution Lens

## Purpose

Inspect the transition from decision or permission into committed action, execution, and observed outcome.

## Questions

- What action was proposed?
- What action was authorized?
- What action was bound?
- What action was executed?
- Did execution match the bound action?
- What prevented unauthorized execution?
- Was refusal possible?
- Was no-bind behavior explicit?
- Was the outcome observed?
- Can the execution path be replayed or reconstructed?

## Pressure Tests

- recommendation != authorization
- authorization != binding
- binding != execution
- execution != intended execution
- execution completed != outcome successful
- signed disposition != executed action
